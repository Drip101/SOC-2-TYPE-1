# Overview

This repository contains a simulated Governance, Risk, and Compliance (GRC) audit engagement for **HealthPulse**, a healthcare data analytics organization operating in AWS cloud infrastructure.

The project was designed to simulate a real-world cybersecurity audit and compliance assessment involving:

* HIPAA Security Rule compliance
* ISO 27001 governance alignment
* AWS cloud security assessments
* Security control validation
* Risk management analysis
* Cybersecurity maturity evaluation

The objective of this project was to assess HealthPulse’s security posture, identify compliance risks, evaluate technical safeguards, and provide actionable recommendations aligned with industry security frameworks.

---

# Project Objectives

The assessment focused on the following objectives:

* Evaluate HIPAA compliance readiness
* Assess ISO 27001 control alignment
* Review cloud security governance within AWS
* Assess identity and access management controls
* Evaluate encryption, logging, and monitoring practices
* Identify operational and compliance risks
* Recommend security improvements

# Key Risks Identified

## Unauthorized Access to PHI

Excessive privileged access and weak identity governance may increase the likelihood of unauthorized access to sensitive healthcare information.

## Cloud Misconfiguration Risk

Improper AWS configurations could expose healthcare data publicly or allow unauthorized access.

## Third-Party Vendor Risk

Vendors with access to PHI may introduce compliance and supply chain security risks.

---

# Cybersecurity Maturity Assessment

HealthPulse demonstrates a moderate cybersecurity maturity level with several foundational controls implemented.

## Strengths

* AWS security controls enabled
* MFA enforced for privileged users
* Security policies documented
* Leadership commitment to ISO 27001 certification

## Areas for Improvement

* Manual access review processes
* Lack of centralized SIEM monitoring
* Immature vendor risk management processes

### Estimated Maturity Level

**Level 2–3 (Developing to Defined)**

---

# Recommendations

## Strengthen Identity & Access Management

* Enforce strict least privilege access
* Automate access review workflows
* Expand MFA enforcement

## Deploy Centralized SIEM Monitoring

Improve:

* Threat detection
* Incident response
* Compliance visibility
* Audit reporting

## Formalize Vendor Risk Management

Implement:

* Vendor security assessments
* Annual vendor reviews
* Business Associate Agreement validation
* Vendor risk scoring
